some advantages [12]. The advantage of paragraph vectors is that paragraph vectors can learn from unlabelled data

2.7. Preprocessing
Preprocessing processes have a huge effect on sentiment analysis. This is because it is closely related to the results
produced. The preprocessing process included several processes, such as stemming, stop word removal, and
tokenizing [8].
Stemming is a process of mapping and removing inflection in a word into the form of a word into base word forms.
There are many approaches to stemming, which depend on what language is used. Stemming process in Indonesia,
for example, will follow the Indonesian rules for affix and others. For example, the word â€œmembelikanâ€ has the affix
â€œmeâ€ and the suffix â€œkanâ€. After going through the process of stemming in the word into base form, it will become
â€œbeliâ€. Stemming can reduce the indexing size by 40-50% [8].
Lemmatization is a task to determine two words that have the same root [13]. Stemming and lemmatization have
the same goal. Both goals are to reduce the form into a base form. Different from stemming that only use algorithm,
lemmatization use complete morphological parsing [13]. The advantage of using lemmatization is getting a more
accurate base word from stemming. The disadvantage of using lemmatization is lemmatization is slower and consumes
more resources than stemming.
Stop word removal is a process of removing words that are not meaningful but are often used repeatedly. These
words are generally in the form of conjunction words that connect between sentences and in a sentence. Words that
fall into the stop word category are 20-30% [8] of a document. Therefore, removing the stop words is expected to
make the model more effective and efficient.
Tokenization is the process of breaking sentences into smaller parts (words). This solving process varies in each
language. For example, in English and Indonesian, the solving process can be broken down based on spaces or
punctuation marks. For other languages, it will follow the rules that apply. An example of tokenization is the following
sentence â€œI eat riceâ€ into an array like this [â€œIâ€,â€ eatâ€,â€ riceâ€]. The purpose of tokenization is to maintain consistency
and identify the meaning of keywords.

3.1. Data collection
In this stage, tweet data is collected through crawling process using the Twitter API using the following words:
- Komodo
- Rinca
- Jurassic Park
Those words were chosen as a keyword because those words are very popular for this topic, like choosing komodo
as a keyword because Rinca Island is the komodo habitat. Rinca is the name of the island where the development will
be located. Jurassic Park because of the development in Pulau Rinca called Jurassic Park by Twitter users. The process
produces 8000 tweet data that contain the keywords. The data collected through Twitter API is then removed from
duplicate and unrelated data. After data cleansing, there are 1038 relevant tweet data.
3.2. Tagging
This process aims to train the model. Each tweet is tagged with a value of -1,0,1. Value 1 is a tweet that supports
or is pro toward the development of Rinca Island. The value 0 is neutral with the development on Rinca Island, or the
tweet is some news. The value -1 is a tweet that rejects or contradicts with development on Rinca Island.
The tagging process is done manually. Three people do a tag with the same tweets data. The final tagging value is
